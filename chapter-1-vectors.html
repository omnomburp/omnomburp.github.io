Chapter 1

Vectors, bases, and matrix transforms

Computer graphics begins with a simple question: how do we describe points and directions in space? Vectors answer that question. Bases tell us what the coordinate axes mean. Matrices tell us how that local frame can rotate, stretch, and, later, with one extra coordinate, translate.

What you should leave with

Confidence reading a point as weighted basis directions rather than a mysterious pair of numbers.

Main idea

A matrix changes how a shape is interpreted inside a coordinate frame.

Why it matters

Everything later in graphics depends on moving data between spaces without losing track of meaning.

Start here p = x i + y j

Read coordinates as weights on basis directions, not as unexplained numbers.

Then change the frame p' = M p

A matrix changes how the same local point is rebuilt in space.

Next question point or direction?

We will learn translation after we separate those two cases clearly.

Mental model

Think of coordinates as instructions for rebuilding a point from basis directions

The numbers inside a vector are not meaningful by themselves. They only become meaningful after you decide what the basis directions are. In graphics, changing the basis is often just as important as changing the point.

1

The raw numbers are not the picture yet

A coordinate like (2, 1) only becomes meaningful after you decide what the basis directions are.

2

A basis picks the local language

The usual 2D basis is “one step right” and “one step up,” but you are free to rotate or stretch that language.

3

Coordinates are basis weights

If a point is (2, 1), read that as “two copies of the x basis plus one copy of the y basis.”

4

A matrix changes that reconstruction rule

Rotation and scale work by changing the basis itself. Translation is special enough that we will slow down and learn it next.

Rebuild A Point

Keep the weights fixed and let the basis move first

Before translation or long matrix chains, keep one sentence in your head: the point is rebuilt from basis directions. The animation below keeps the weights fixed while the basis slowly rotates and stretches.

Visual explainer

Same weights every frame. Different basis every frame. The point is rebuilt from the moving basis.

  1. Start with one local coordinate

    Imagine the point (1.2, 0.65). Read that as “take 1.2 copies of i and 0.65 copies of j.”

  2. Build the point from those two contributions

    The orange segment shows the contribution from i. The green segment adds the j contribution to reach the final point.

  3. Move the basis, not the weights

    As the basis rotates and stretches, the same weights rebuild a different final position. That is the core visual idea behind a matrix transform.

Plain-language sentence

“The point moved because the frame I used to rebuild it changed.”

What to notice

The weights stay constant throughout the animation. Only the basis directions and their lengths are changing.

Translation And W

Translation needs one extra coordinate because points and directions should not behave the same way

Rotation and scale can be understood as basis changes. Translation is different: it should move points, but it should not move pure directions. Graphics handles that by adding one extra coordinate called w.

Visual explainer

Same local value, same translation vector. The point moves because w = 1; the direction stays put because w = 0.

  1. A point should move with translation

    If a point marks a location on an object, translating the object should move that location too.

  2. A direction should keep its meaning

    If a value means “this way,” shifting the whole object should not change that direction or its length.

  3. Homogeneous coordinates add one more slot

    Write points as (x, y, z, 1) and directions as (x, y, z, 0). Now the matrix chain can tell those cases apart.

  4. The translation offset is multiplied by w

    When w = 1, the offset is applied. When w = 0, the translation contribution disappears.

Point p_h = (x, y, z, 1)
Direction d_h = (x, y, z, 0)

Order Matters

Applying the same transforms in a different order produces a different motion

This idea is easier to trust once you see it. Both panels below use the same changing scale and rotation values, but the highlighted corner traces a different path because the second transform always acts on the result of the first.

Visual explainer

Left: scale then rotate. Right: rotate then scale. The highlighted corner follows a different orbit in each case.

Scale then rotate

The shape stretches along its local axes first, then the already-stretched result is turned as a whole.

Rotate then scale

The shape turns first, so the later scale happens along axes that have already been rotated.

Why this matters

Graphics pipelines constantly mix points, directions, normals, and offsets. This is one of the first places where confusing those meanings causes bugs.

  • Drag the orange point first. The readout is solving for basis weights, not screen coordinates.
  • Turn translation off and notice that the local reconstruction stays, but the offset no longer applies.
  • Turn translation back on and read the result as “basis reconstruction first, affine offset second.”

Drag the orange local point. The cyan copy only moves when the value is treated like a point with w = 1.
