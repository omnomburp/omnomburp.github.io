Chapter 3

Surface normals explain why light changes across a shape

Shading gets much easier once you stop thinking in terms of “this triangle should be bright” and start thinking in terms of “this surface is facing the light.” Surface normals provide that facing direction.

Core skill

Read a normal as an orientation vector attached to the surface.

Lighting link

Use the dot product between a normal and a light direction to measure alignment.

Debugging habit

If lighting looks wrong, inspect normals before blaming the material.

Diffuse light brightness = max(dot(n, l), 0)
Perfect alignment dot(n, l) = 1
Facing away dot(n, l) <= 0

Game Intuition

A flashlight sliding across armor is a normal-direction story

In a game, a shield, helmet, wall, or character shoulder can stay in the same place while its brightness changes dramatically. The missing ingredient is usually the surface normal, not a mystery light rule.

A moving flashlight hits a curved shield. The highlighted point changes brightness because the local normal and incoming light direction align differently over time.

Surface normal

The white arrow shows which way that tiny patch of armor is facing.

n

Light direction

The orange arrow points from the surface point toward the flashlight.

l

Alignment test

The brighter patch is the one whose normal lines up best with the light.

dot(n, l)

Game effect

This is why a curved surface gets a moving highlight instead of one flat brightness value.

diffuse = max(dot(n, l), 0)

  1. The flashlight is not asking “where is the shield?”

    It already knows where the surface point is. The shading question is which way that point is facing.

  2. Different points on the same object can face the light differently

    That is why a round shield or shoulder pad gets a bright hotspot that slides instead of one uniform color.

  3. The normal changes the answer even when the light stays fixed

    Move to a different patch on the curved surface and the angle to the flashlight changes, so the diffuse term changes too.

  4. This same rule drives many game materials

    Characters, props, terrain, and walls all start from this same normal-versus-light alignment test before more advanced shading layers are added.

Useful debug habit

If a highlight appears in the wrong place on a model, check the normals before tweaking light intensity or material values.

Why chapter 3 matters

Once this picture is clear, shader code stops looking like mysterious dot-product folklore and starts reading like visible geometry logic.

Mental model

A normal is not where the surface is. It is how the surface is oriented.

1

Normals are directions

They point away from the surface and tell you which way that part of the shape is facing. Many mesh normals begin as cross(edge1, edge2).

2

Light has a direction too

To decide how much light arrives, compare the light direction to the surface normal.

3

The dot product measures alignment

If two vectors point together, the value is large. If they point apart, the value drops.

4

Interpolated normals smooth the look

Normals can vary across a triangle, which makes a faceted mesh appear smoothly shaded.

Step by step

How the lighting term is built

  1. Normalize the normal

    Use a unit-length direction so the dot product reports alignment instead of getting mixed up with length.

  2. Normalize the light direction

    Now both vectors talk about orientation in the same way.

  3. Take the dot product

    This gives a value from about -1 to 1, depending on how aligned the vectors are.

  4. Clamp negative values

    Surfaces facing away from the light should not receive positive diffuse light, so values below zero are clipped.

Non-uniform scale warning

Normals are directions, not positions. Under non-uniform scale, they need a special transform called the inverse transpose.

Why visualizing helps

When you draw the normals as lines, lighting bugs often become obvious immediately.

What this unlocks next

Once you know what data a shader needs, the split between vertex and fragment work becomes much easier to follow.

When the bug appears

Pure rotation keeps the normal correct, so the fix looks unnecessary. The error shows up once the scale is uneven.

What the fix means

The inverse transpose is not magic. It is the linear-algebra move that keeps a normal perpendicular to the transformed surface.

Why the meter helps

If the patch orientation looks right but the meter drops unexpectedly, the normal pipeline is the first place to inspect.

  • Turn non-uniform scale on and fix off. The patch is the same surface, but the shader normal tilts incorrectly.
  • Turn the fix back on and compare the numeric dot product before and after. That is the bug becoming measurable.
  • Use this chapter whenever a lit object looks wrong even though the geometry appears to be in the right place.
