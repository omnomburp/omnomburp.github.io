Chapter 10

Ray tracing and rasterization

This chapter compares two rendering mindsets so you can place all the earlier ideas in the broader graphics landscape.

On this page

Mental model Rasterization flow Ray tracing flow Comparison

Study prompt

Do not ask which method is “better” in the abstract. Ask which visibility and lighting problem each method is solving well.

Chapter 10

Rasterization projects triangles. Ray tracing follows rays through space.

Both methods aim to produce an image, but they begin from different questions. Rasterization asks which triangles cover each screen region after projection. Ray tracing asks which object a ray from the eye hits first, then often asks more visibility questions from that hit point.

Rasterization strength

Very efficient for visible-surface rendering of triangle meshes, which is why it dominates real-time graphics.

Ray tracing strength

Natural for reflections, shadows, and recursive light transport because rays can ask direct visibility questions.

Shared reality

Both still need scene data, acceleration ideas, material models, and careful numerical handling.

Mental model

These are different ways of asking “what should this pixel show?”

1

Rasterization starts with geometry projected onto the screen

It works forward from triangles toward pixels, which makes visible-surface rendering very fast.

2

Ray tracing starts with a viewing ray

It works from the eye into the scene, asking what object each ray hits and what light reaches that hit point.

3

Rasterization needs extra tricks for many lighting effects

Shadow maps, reflection probes, and screen-space effects are common because those answers are not built into the basic method.

4

Ray tracing pays more per visibility question

Its lighting logic can be more direct, but the intersection work is much more expensive unless heavily optimized.

Rasterization flow

How the pipeline you have learned so far fits together

  1. Transform vertices through the usual spaces

    Model, view, and projection move triangles into clip space.

  2. Rasterize the projected triangles

    The GPU determines screen coverage and generates fragments.

  3. Use interpolated data for shading

    Normals, UVs, and colors are blended across the triangle and used in fragment shading.

  4. Resolve visibility with depth

    The nearest fragment usually wins for each pixel-sized region.

Why games love it

It is highly optimized for triangles and predictable frame times, which matters for interactive applications.

What it struggles with

Global illumination, accurate reflections, and long chains of visibility questions often need approximations.

Ray tracing flow

How the ray-based mindset changes the problem statement

  1. Shoot a primary ray for the pixel

    The ray starts at the eye and moves through the pixel into the scene.

  2. Find the nearest intersection

    The renderer determines which object the ray reaches first.

  3. Shade the hit point

    At that point, the renderer can ask secondary visibility questions: can the light see this point, what does it reflect, what refracts through it?

  4. Accumulate lighting contributions

    Multiple rays and multiple bounces produce more realistic light transport, at a steep computational cost.

Why acceleration matters

Testing every ray against every triangle would be too expensive, so ray tracers rely on spatial acceleration structures.

Why noise appears

Path tracing often estimates lighting with random samples, so too few rays produce noisy images that need denoising or more sampling.

Comparison

Use this table to keep the two approaches mentally separate

Rasterization asks

Which projected triangle fragments cover this region of the screen, and which of those is closest?

Ray tracing asks

Which object does this ray hit first, and what additional rays should be spawned from that interaction?

Real systems use hybrids

Modern engines often rasterize primary visibility and use ray tracing selectively for reflections, shadows, or global illumination features.

Good summary sentence

Rasterization is a projection pipeline for triangles; ray tracing is a visibility-query pipeline for rays.

What the beginner should retain

Neither method removes the need for good material models, coordinate reasoning, or color handling.

Final chapter connection

Whatever method forms the image, it still has to survive exposure, display transfer, and post-processing before it reaches the viewer.
