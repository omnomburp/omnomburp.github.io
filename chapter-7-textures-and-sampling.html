Chapter 7

Textures and sampling

This chapter explains how image data gets attached to a surface and why the sampling rule matters just as much as the texture itself.

Study prompt

Whenever a texture looks noisy or blocky, ask whether the problem is the image itself or the way it is being sampled.

Chapter 7

Textures are sampled data attached to geometry through UV coordinates

Textures are not pasted onto geometry by magic. Each vertex carries UV coordinates that describe how to look up values inside a texture image. Then the GPU samples between those texels according to a filtering rule.

UV coordinates

These are 2D lookup coordinates that travel with the mesh and tell the shader where to sample.

Filtering

Nearest keeps hard texel boundaries. Linear blends nearby texels and looks smoother.

Sampling artifacts

Aliasing appears when a dense pattern is projected onto too few pixels without enough filtering support.

Mental model

A texture is just a table of samples until a surface tells the GPU how to read it

1

The mesh carries UV coordinates

Every vertex can store a matching point in texture space so the surface knows what part of the image it should use.

2

Rasterization interpolates UVs

Fragments inside a triangle receive UV values blended from the triangle’s vertices.

3

The shader samples the texture

That UV coordinate is used to fetch or blend texel values from the texture image.

4

Filtering controls the visual tradeoff

Sharp texel edges can look crisp or noisy. Blended filtering looks smoother but can soften detail.

Sampling story

A single pixel asks for a UV, then the sampler decides which texels are allowed to answer

Texture sampling becomes easier to reason about when you stop imagining the whole image at once. One fragment gets one interpolated UV, and the GPU has to turn that continuous coordinate into a discrete texel result.

Visual explainer

The moving fragment inherits a UV inside the triangle, then nearest and linear filtering turn that same lookup into different output colors.

  1. The triangle corners start with known UV coordinates

    Each vertex anchors a place in texture space, just like it anchors a position in model space.

  2. Rasterization interpolates a UV for the fragment you are shading

    The fragment does not inherit a texel directly. It inherits a blended UV location somewhere inside the texture.

  3. The sampler lands between texels on a discrete grid

    That is why textures are not continuous functions. The GPU has to interpret a fractional coordinate against stored samples.

  4. Filtering decides whether one texel wins or nearby texels blend

    Nearest chooses the closest texel. Linear blends the surrounding texels. Mipmaps help when one screen pixel covers a much larger footprint.

Texel versus pixel

A texel lives in the texture image. A pixel lives on the display. Sampling is the bridge between those two grids.

Why shimmer happens

If one screen pixel covers many texels, the sampler is being asked to summarize too much detail without enough prefiltering support.

Step by step

How a texture sample reaches the final image

  1. Author or generate UVs

    The mesh needs a mapping from its surface into 2D texture space.

  2. Pass UVs from the vertex shader

    The vertex shader forwards them as varyings so the fragment stage can see them.

  3. Sample the texture in the fragment shader

    The fragment shader asks for the texture value at the interpolated UV location.

  4. Filter according to scale and mode

    When the surface is steep or far away, one on-screen pixel may cover many texels, which is where sampling problems begin.

What mipmaps solve

If the texture pattern gets much smaller than a pixel, prefiltered mip levels help stop high-frequency flicker.

Why UV seams happen

A 3D surface often has to be cut apart when flattened into 2D texture space, just like an orange peel.

Color textures versus data textures

Base-color images are usually authored for display and often need sRGB decoding before lighting. Normal, roughness, and mask textures are data and should stay linear.

What comes next

Materials decide how sampled textures, normals, and light directions combine into a final shaded surface.

Mipmap story

When one screen pixel covers many texels, the sampler should switch to a smaller prefiltered image

Filtering between nearby texels is not enough once the surface gets far away or highly tilted. At that point a single screen sample is standing in for a whole patch of the texture, and the stable answer is usually to read from a smaller mip level that was prefiltered ahead of time.

Visual explainer

As the texture footprint grows, the chosen mip level gets smaller and more prefiltered so the sampled answer stays stable.

  1. The base texture can contain more detail than the screen can show

    Once the projected footprint grows, one pixel is no longer asking about one texel-sized neighborhood.

  2. Mipmaps store prefiltered copies at lower resolutions

    Each smaller level summarizes a wider area, which gives the sampler a better starting point for minification.

  3. The sampler chooses a level based on footprint size

    A larger footprint means the stable answer usually comes from a lower-resolution mip level.

  4. That is why mipmaps reduce shimmer

    They replace unstable high-frequency guessing with a controlled prefiltered summary of the same texture.

Magnification versus minification

Nearest and linear mainly help when you are close enough to care about nearby texels. Mipmaps matter when you are far enough that many texels collapse into one pixel.

Seams still matter

Mipmaps solve prefiltering, not bad UV layout. A seam is still a discontinuity in how the surface was unwrapped.

Live lab

Change UV frequency and filtering on a textured plane

This plane uses a procedural checker texture. Increase the UV repeat count to force more texture detail onto fewer pixels. Then switch between nearest and linear filtering and notice how the texture either holds hard texel boundaries or blends them.

  • Increase UV repeat until the checker becomes dense. That forces the sampler to work harder.
  • Lower the tilt so the plane becomes more grazing to the camera. That increases minification artifacts.
  • Toggle linear filtering and note that it smooths the pattern but also softens the texel boundaries.

The texture is the same in every case. The visible difference comes from the sampling rule.
